## 1. Host Setup

### 1.1 Proxmox Installation

Install Proxmox VE 9.x from ISO. During install, configure management interface on the physical NIC. After install, WiFi (`{{WIFI}}`) becomes the uplink — the ethernet NIC (`{{NIC}}`) is unused and has no carrier.

### 1.2 PATH Fix

`{{PATH}}` is not in PATH for root SSH sessions. Add permanently:

```bash
echo 'export PATH=$PATH:{{PATH}}' >> /root/.bashrc
source /root/.bashrc
```

Affected commands without this fix: `ifreload`, `iptables`, `pvesm`, and others.

### 1.3 NAT Bridge ({{BRIDGE_NAME}})

hp600a is WiFi-only. Linux bridges cannot enslave WiFi interfaces directly. A NAT bridge with no physical port is used instead.

Create bridge via Proxmox UI: **Node → Network → Create → Linux Bridge**

| Field | Value |
|---|---|
| Name | `{{BRIDGE_NAME}}` |
| IPv4/CIDR | `{{BRIDGE_IP}}` |
| Bridge ports | *(empty)* |
| Autostart | ✅ |
| Comment | `DMZ NAT bridge` |

Then edit `/etc/network/interfaces` and add `post-up` lines to the `{{BRIDGE_NAME}}` block:

```
auto {{BRIDGE_NAME}}
iface {{BRIDGE_NAME}} inet static
    address {{BRIDGE_IP}}
    bridge-ports none
    bridge-stp off
    bridge-fd 0
    post-up echo 1 > /proc/sys/net/ipv4/ip_forward
    post-up iptables -t nat -A POSTROUTING -s '10.10.10.0/24' -o {{WIFI}} -j MASQUERADE
    post-down iptables -t nat -D POSTROUTING -s '10.10.10.0/24' -o {{WIFI}} -j MASQUERADE
```

Apply:

```bash
ifreload -a
```

Verify:

```bash
ip a show {{BRIDGE_NAME}}
iptables -t nat -L -n -v
```

### 1.4 NFS Storage

Proxmox web UI cannot add NFS shares with `--mkdir 0`. All NFS shares on hp600a must be added via CLI.

**Read-only shares (e.g. ISOs):**

```bash
pvesm add nfs ISOs \
  --server {{TS_NAS_IP}} \
  --export /mnt/Storage/Soft/MINILAB/ISOs \
  --path /mnt/pve/ISOs \
  --content iso \
  --mkdir 0
```

**Backup shares:**

```bash
pvesm add nfs Backups \
  --server {{TS_NAS_IP}} \
  --export /mnt/Storage/Soft/MINILAB/hp600a \
  --path /mnt/pve/Backups \
  --content backup \
  --mkdir 0

mkdir /mnt/pve/Backups/dump
```

> **Note:** Always use TrueNAS Tailscale IP (`{{TS_NAS_IP}}`), not the hostname `{{TS_NAS_NAME}}`. The Proxmox storage daemon (`pvestatd`) resolves hostnames independently of Tailscale and will report the storage as inactive if the hostname is used.

> **Note:** If storage shows inactive after applying network changes, check `pvestatd` can reach TrueNAS: `rpcinfo -T tcp {{TS_NAS_IP}} nfs 4`

### 1.5 ifupdown2

Required for `ifreload -a`. Installed by default on PVE 9 but verify:

```bash
dpkg -l ifupdown2
```

If missing:

```bash
apt update && apt install ifupdown2
```

## 2. Day-2 Operations — Proxmox Host

### 2.1 SSH Access

```bash
ssh {{SSH_USER}}@hp600a
```

Via Tailscale only. No direct SSH on guest WiFi.

### 2.2 Proxmox Web UI

```
{{DMZ_BOX_URL}}
```

Via Tailscale only.

### 2.3 Check Node Health

```bash
pvesm status
ip a show {{BRIDGE_NAME}}
iptables -t nat -L -n -v
tailscale status
```

### 2.4 Taking a Backup (VM Snapshot to NFS)

Backups require temporary TrueNAS ACL access. Follow this procedure:

1. On TrueNAS: open ACL for `{{TS_DMZ_IP}}` on the `hp600a` NFS share
2. Run backup from Proxmox UI: **VM → Backup → Backup Now** (storage: Backups)
3. Wait for backup to complete
4. On TrueNAS: close ACL

> **Future:** Replace with USB drive attached directly to hp600a to eliminate ACL dependency.

### 2.5 NFS Storage Not Online

If `pvesm status` shows NFS storage as inactive despite the mount being live:

```bash
rpcinfo -T tcp {{TS_NAS_IP}} nfs 4
mount | grep 100.87
```

If mount is present but pvestatd reports inactive, restart the daemon:

```bash
systemctl restart pvestatd
pvesm status
```

If storage is missing from `storage.cfg` but mount exists, re-add via CLI (see 1.4).

### 2.6 Removing a Stale NFS Mount

```bash
umount -f -l /mnt/pve/<name>
rmdir /mnt/pve/<name>
```

## 3. DMZ VM (VM 100)

**Name:** DMZ
**OS:** Ubuntu Server 24.04.3
**Tailscale IP:** `100.99.139.25`
**Bridge:** `{{BRIDGE_NAME}}` (NAT, `{{BRIDGE_GATEWAY}}0/24`)
**Docker:** CLI only (no Portainer agent — see ADR-0021)

### 3.1 SSH Access

```bash
ssh {{SUDO_USER}}@dmz
```

Via Tailscale SSH only. openssh-server is removed. No direct SSH on `{{BRIDGE_GATEWAY}}0`.

### 3.2 Verify VM Health

```bash
tailscale status
docker ps
sudo ss -tlnp | grep 22
```

Port 22 should show only `tailscaled`. No `sshd`.

### 3.3 Deploy / Update a Docker Container

```bash
cd /path/to/module
docker compose pull
docker compose up -d
```

### 3.4 View Container Logs

```bash
docker logs <container_name>
docker compose logs -f
```

### 3.5 Taking a Snapshot

Before any major change:

1. In Proxmox UI: **VM 100 → Snapshots → Take Snapshot**
2. Name: descriptive (e.g. `pre-ghost-deploy`)

### 3.6 Rebuilding the DMZ VM from Scratch

The DMZ VM is stateless and disposable. To rebuild:

1. In Proxmox: destroy VM 100
2. Create new VM following setup steps (see closure notes or re-run from issue)
3. Install Ubuntu Server 24.04, static IP `{{BRIDGE_GATEWAY}}0/24`, gateway `{{BRIDGE_GATEWAY}}`
4. Install Tailscale, register node, apply ACLs
5. Disable openssh: `sudo systemctl disable --now ssh && sudo kill $(pgrep sshd)`
6. Install Docker: `curl -fsSL https://get.docker.com | sh`
7. Install qemu-guest-agent: `sudo apt install qemu-guest-agent -y`
8. Enable QEMU Guest Agent in Proxmox VM Options
9. Redeploy services via Docker CLI
10. Take baseline snapshot

### 3.7 Tailscale SSH Not Working

```bash
sudo tailscale status
sudo tailscale set --ssh
sudo ss -tlnp | grep 22
```

If sshd is running unexpectedly:

```bash
sudo systemctl disable --now ssh
sudo kill $(pgrep sshd)
```

### 3.8 Internet Connectivity Lost from DMZ VM

Check NAT rule is present on hp600a:

```bash
iptables -t nat -L -n -v | grep 10.10.10
```

If missing, re-apply:

```bash
iptables -t nat -A POSTROUTING -s '10.10.10.0/24' -o {{WIFI}} -j MASQUERADE
```

Persistent fix: verify `post-up` lines are present in `/etc/network/interfaces` and run `ifreload -a`.
